# pdfocr2 日本語マニュアル

## 1. 概要

`pdfocr2` は、スキャンされた PDF や画像を OCR で読み取り、検索可能 PDF として保存できる Windows デスクトップアプリです。

主な用途:

- 紙をスキャンした PDF の文字起こし
- 画像化された文書のテキスト抽出
- OCR 結果を確認しながら修正
- 複数ファイルのまとめ処理

## 2. できること

- PDF / 画像ファイルを開く
- PaddleOCR で OCR を実行する
- 必要に応じて VLM で認識結果を補正する
- テキストやボックス位置を GUI 上で修正する
- 検索可能 PDF として書き出す
- 複数ファイルをバッチ処理する

## 3. 使い始める前の準備

1. 配布ファイル `pdfocr2-package-win64.7z.001` と `pdfocr2-package-win64.7z.002` を同じフォルダに置きます。
2. `pdfocr2-package-win64.7z.001` を 7-Zip で開き、任意のフォルダへ展開します。
3. 展開先で `pdfocr2.exe` を実行します。

注意:

- 展開後のファイルは同じフォルダにまとめたまま使ってください。
- 書き込み可能なフォルダに展開してください。

## 4. 基本的な使い方

1. `pdfocr2.exe` を起動します。
2. スキャン PDF または画像ファイルを開きます。
3. OCR を実行します。
4. 認識結果を確認し、必要なら修正します。
5. 検索可能 PDF として保存します。

## 5. OCR の流れ

通常の流れは次の通りです。

1. ファイルを読み込む
2. ページ画像を解析する
3. OCR で文字を認識する
4. 必要なら補正する
5. 出力する

スキャン品質が高いほど認識結果も安定します。

## 6. 編集

OCR 後は GUI 上で内容を見直せます。

- 認識されたテキストの修正
- ボックス位置や範囲の見直し
- ページごとの確認

認識ミスが気になる箇所だけ手で直してから PDF 出力できます。

## 7. VLM 補正

VLM 補正は任意機能です。設定しなくても通常 OCR は使えます。

利用方法:

- ローカル利用: LM Studio
- 外部 API: OpenAI / Gemini / Claude

VLM は、通常 OCR では崩れやすい複雑な文字配置や読みにくい部分の補助に向いています。

## 8. GPU 利用

CUDA 対応環境では GPU OCR を利用できます。

- GPU が使える場合は高速化が期待できます。
- GPU 初期化に失敗した場合は CPU に自動フォールバックします。

GPU がなくても基本機能は利用できます。

## 9. バッチ処理

複数の PDF や画像をまとめて処理できます。

向いている用途:

- 大量のスキャン PDF の一括 OCR
- 定型的な文書のまとめ処理

処理前に少数ファイルで結果を確認してから本番実行するのが安全です。

## 10. 出力

主な出力は検索可能 PDF です。

用途:

- Acrobat などで文字検索できる PDF を作る
- 元の見た目を残したままテキスト検索を可能にする

## 11. うまくいかないとき

次を確認してください。

- 配布ファイルをすべて展開しているか
- `pdfocr2.exe` と同梱ファイルを別々にしていないか
- 入力ファイルが極端に低解像度ではないか
- GPU 利用時に CUDA 環境が有効か
- VLM 利用時に LM Studio または API 設定が正しいか

## 12. 補足

- この配布物は Windows 向けです。
- VLM 機能は別途プロバイダ設定が必要です。
- OCR 結果は元画像の品質に影響されます。
